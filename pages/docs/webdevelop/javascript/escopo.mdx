# Escopo

O escopo é a região do código onde uma determinada variável** é acessível ou visivel**.

O escopo define o **contexto** no qual uma variável pode ser referenciada e modificada.

Tente pensar no contexto como cômodos de uma casa. Por excemplo, para utilizar o fogão você deve estar na cozinha.

# Tipos de escopo

- **Global:** variáveis declaradas fora de qualquer função ou bloco de código(var).
  excemplo:

```javascript
var mensagem = "Olá mundo!"; // Variável global

function mostrarMensagem() {
  console.log(mensagem); // Pode acessar a variável global
}
mostrarMensagem(); // Imprime: Olá mundo!
```

- **Bloco:** acessíveis apenas dentro do bloco de código onde foram declaradas(let e const).

- **Local (ou escopo de função):** variáveis declaradas dentro de uma função.

# Hoisting

Hoisting (**levantar ou içar**) se refere ao compontamento do interpretador de **mover as declarações** de variáveis e funções para o **topo do escopo em que foram definidas**, antes mesmo da execução do código.

Esse comportamento possibilita usar uma variável ou função antes que ela esteja definida.

## Hoisting de variáveis

Todas as declarações de variáveis são movidas para o topo do seu escopo independentemente de onde tenha sido declarada, ela estará disponível para uso em todo escopo em que foi definida.

**Importante:** mesmo que as declarações de variáveis sejam movidas para o topo do escopo, elas ainda precisam ser declaradas antes de serem utilizadas. Caso contário, você receberá uma ferência indefinida(undefined).

## Hoisting de funções

Todas as declarações de funções também são movidas para o topo do seu escopo. Isso significa que você pode chamar uma função antes mesmo de declará-la.

Essa caractéristica do javascript permite que você organize seu código de forma mais intuitiva, definindo as funções em qualquer ordem que desejar.

| keyword           | const | let | var |
| ----------------- | ----- | --- | --- |
| global scope      | No    | No  | yes |
| function scope    | Yes   | Yes | Yes |
| block scope       | Yes   | Yes | No  |
| can be reassigned | No    | Yes | Yes |

A tabela acima compara as principais características das palavras-chave `const`, `let` e `var` em JavaScript:

- **global scope**: Indica se a variável tem escopo global. Apenas `var` pode ter escopo global.
- **function scope**: Indica se a variável tem escopo de função. Todas as três palavras-chave têm escopo de função.
- **block scope**: Indica se a variável tem escopo de bloco. `const` e `let` têm escopo de bloco, mas `var` não.
- **can be reassigned**: Indica se a variável pode ser reatribuída. `let` e `var` podem ser reatribuídas, mas `const` não.

<Callout type="warning">
  ### Atenção
  É importante notar que, embora o hoisting ocorra com var, seu uso
  tem algumas desvantagens, como a falta de escopo de bloco adequada e a
  possibilidade de levar a comportamentos inesperados.
</Callout>
Devido a essas desvantagens, **recomenda-se o uso let e const** em vez de var, pois eles têm escopo de bloco e não são içados da mesma maneira que var.
